FSL Mobile app – Offline Compatibility


  Sahil  /  July 15, 2024  /  Salesforce, Field Service Mobile, Salesforce Field Service, FSL Mobile App Offline


How to use Salesforce Field Service Mobile app in offline mode?

Many a times, there will be a requirement to the field service technician where they do not have access to internet and that’s when they would need to use the FSL Mobile app to access the data and need to utilize the functionality set up in the FSL Mobile flows, and, to make these changes in the relative objects like Product requirements or Work Order or Service Appointments etc.


To achieve this, salesforce has provided “Briefcase Builder” which will make FSL Mobile app to be used in offline mode.


Let’s understand this step by step:

First, Let's see what is a Briefcase and Briefcase Builder?

A briefcase is a set of rules and filters that select the records to be used in offline mode.

A Briefcase Builder, where you can create and manage your organization’s briefcases. By connecting a briefcase with a compatible mobile app, you can enable your mobile workforce to access the most relevant records, even while offline.

    Some Notes to remember:
  • Briefcase Builder supports up to 50,000 records that can be retrieved offline.
  • Briefcase Builder works with both standard and custom Salesforce objects.
  • With a briefcase, rules/filters can be created for Prime and related objects which are accessible in offline mode.

Now, Let's take a look at how to create a BriefCase:

  • From Setup, in the Quick Find box, enter Briefcase Builder, and then select Briefcase Builder.
  • Click New Briefcase.
  • Enter a name and an optional description for your briefcase and Click Next



  • Select the Object to add and set up rules to configure to identify the objects that are included in the briefcase along with related object.



  • Related object can be added along with main/prime object along with their own separate rules/filter criteria.



  • Custom metadata types are also available to add optionally with custom metadata rules to the briefcase. We can use custom metadata rule in the briefcase to support a custom flow in the Salesforce Field Service mobile app.

    • For more information on custom metadata rules, see Data Selection.


  • We can assign users or user groups to this briefcase which will determine who can access the records in the briefcase.

      • Some points to consider :

    • Only user groups are supported. Chatter groups, or collaboration groups, aren’t supported.
    • Multiple briefcases can be assigned to same users.
      Note that, Assigning multiple briefcases to one user will give them access to all the offline data combined from all the assigned briefcases.
    • If a user doesn’t have access to the field used to define a dataset, then that user can’t access any records for that dataset.




  • Select the connected apps that you want to associate with this briefcase.



  • To save and activate the briefcase, click Activate. To save the briefcase’s settings without activating it for use, click Save.

Voilà – FSL Mobile App is ready to be used in offline mode.
